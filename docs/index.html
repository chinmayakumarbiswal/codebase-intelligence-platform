🚀 Codebase Intelligence

Intelligent Code Analysis with AI-Powered RAG

Project Overview

Codebase Intelligence is an advanced RAG (Retrieval-Augmented Generation) system that enables intelligent code analysis through conversational AI. Ask questions about your codebase and get contextually relevant answers powered by local LLM inference.

Key Features

📊 Code Indexing

Automatically clone repositories and index 20+ programming languages with intelligent code chunking.

🧠 AI-Powered Search

768-dimensional vector embeddings for semantic similarity search across your entire codebase.

💬 Chat Interface

Natural language queries with source code retrieval and contextual answers from LangChain RAG.

🔒 Privacy First

Local LLM inference with Ollama - no data sent to external APIs.

⚡ Real-time Progress

Live tracking of indexing progress with file count and embedding status updates.

🎯 Multi-Project

Manage multiple repositories with project isolation and concurrent indexing support.

System Architecture

Three-Tier Architecture

Codebase Intelligence follows a three-tier architecture pattern with microservices for separation of concerns:

Presentation Layer (Frontend)

  • React 18 with Material-UI components
  • Vite for fast development and production builds
  • Real-time progress tracking with 500ms polling
  • Zustand for state management (auth, conversations)

Application Layer (Backend)

  • Express.js REST API on port 3001
  • LangChain RAG orchestration engine
  • JWT authentication with OAuth2 integration
  • Background indexing with progress tracking

Data Layer (Storage & Services)

  • MySQL: Persistent data (users, projects, conversations)
  • Chroma: Vector embeddings with similarity search
  • Ollama: Local LLM inference (tinyllama + nomic-embed-text)

Data Flow - Chat Query

User Query ↓ [Frontend: Send query + JWT] ↓ [Backend: Validate JWT] ↓ [RAG Pipeline] ├─→ [Retrieve: Similarity search in Chroma] ├─→ [Rerank: Select top chunks] └─→ [Generate: Send to Ollama with context] ↓ [Return response with source chunks] ↓ [Display in chat interface]

Request Timeout Configuration

Component Timeout Reason
Frontend Axios 300 seconds (5 min) Ollama inference can take 60+ seconds
Ollama Inference No hard limit Process until complete
MySQL Connection 30 seconds Pool timeout for idle connections

Security Architecture

  • Authentication: JWT tokens with OAuth2 integration (GitHub/Google)
  • Authorization: User-scoped project access control
  • Data Isolation: Conversations isolated per project per user
  • Database: SQL injection prevention with parameterized queries
  • Privacy: All LLM inference local (no external API calls)

Database Architecture

Database Schema

Database: MySQL 8 | Name: codebase_intelligence

Core Tables

1. Users Table

Stores user account information and OAuth credentials.

CREATE TABLE users (
  id VARCHAR(36) PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  full_name VARCHAR(255),
  oauth_provider VARCHAR(50),
  oauth_id VARCHAR(255) UNIQUE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

2. Projects Table

Stores cloned repository metadata and indexing status.

CREATE TABLE projects (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  name VARCHAR(255) NOT NULL,
  repo_url VARCHAR(2048) NOT NULL,
  storage_path VARCHAR(2048) NOT NULL UNIQUE,
  status ENUM('cloning', 'indexing', 'ready', 'error'),
  file_count INT,
  chunk_count INT,
  indexed_at TIMESTAMP NULL,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

3. Conversations Table

Stores chat session metadata.

CREATE TABLE conversations (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  project_id VARCHAR(36) NOT NULL,
  title VARCHAR(500),
  message_count INT DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (project_id) REFERENCES projects(id)
);

4. Messages Table

Stores chat messages with source code chunks.

CREATE TABLE messages (
  id VARCHAR(36) PRIMARY KEY,
  conversation_id VARCHAR(36) NOT NULL,
  role ENUM('user', 'assistant') NOT NULL,
  content LONGTEXT NOT NULL,
  source_chunks JSON,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (conversation_id) REFERENCES conversations(id)
);

Entity Relationships

  • User → Projects: 1:N (One user has many projects)
  • User → Conversations: 1:N (One user has many conversations)
  • Project → Conversations: 1:N (Project has many chats)
  • Conversation → Messages: 1:N (Chat has many messages)

Indexing & Performance

Table Index Purpose
users idx_email, idx_oauth Fast user lookup
projects idx_user_id, idx_status Filter by user/status
conversations idx_user_id, idx_project_id User conversations
messages idx_conversation_id Get conversation messages

# Copy environment template

# Edit with your configuration

  • GITHUB_CLIENT_ID & GITHUB_CLIENT_SECRET

# Windows PowerShell

docker-compose down
docker rmi docker-frontend docker-backend -f
docker-compose up --build -d

Step 4: Verify Installation

# Check all containers running

# Verify services

Troubleshooting

Port Already in Use

If port 3000, 3001, etc. are already in use:

# Find process using port
lsof -i :3000  # macOS/Linux
netstat -ano | findstr :3000  # Windows

# Edit docker-compose.yml to use different ports
# Change: "3000:3000" to "3001:3000"

MySQL Connection Failed

# Check MySQL logs
docker logs codebase_mysql

# Verify credentials match .env
# Restart MySQL
docker-compose restart mysql

Ollama Models Not Found

# Pull required models
docker exec codebase_ollama ollama pull tinyllama
docker exec codebase_ollama ollama pull nomic-embed-text

# Verify
curl http://localhost:11434/api/tags

API Reference

Authentication & Base Info

All API endpoints (except public ones) require JWT authentication via Authorization header:

Authorization: Bearer {JWT_TOKEN}

Base URL: http://localhost:3001/api

Response Format: JSON

1. Authentication Endpoints

POST /auth/login

Authenticate user with email and password.

Body:
{
  "email": "user@example.com",
  "password": "password123"
}

Response:
{
  "token": "eyJhbGc...",
  "user": {
    "id": "user-uuid",
    "email": "user@example.com",
    "fullName": "John Doe"
  }
}

POST /auth/github

OAuth login with GitHub (returns redirect URL).

Response:
{
  "redirectUrl": "https://github.com/login/oauth/authorize?client_id=..."
}

GET /auth/callback

OAuth callback handler. Redirects to /projects on success.

Query Params: ?code=github_auth_code
Response: Redirects to /projects (or /login on error)

GET /auth/profile

Get current user profile (requires auth).

Response:
{
  "id": "user-uuid",
  "email": "user@example.com",
  "fullName": "John Doe",
  "createdAt": "2024-01-01T00:00:00Z"
}

POST /auth/logout

Logout user and invalidate token.

Response:
{
  "message": "Logged out successfully"
}
